title: "Webpack common modules"
description: "Pre-resolved Discord internal modules available from @webpack/common — React, Flux stores, UI components, and utilities."
---

Discord ships React, its Flux stores, and a large collection of utilities as part of its webpack bundle. Vencord exposes these through a single import path so you do not have to locate them yourself.

```typescript
import { UserStore, React, Tooltip, FluxDispatcher } from "@webpack/common";
```

All exports from `@webpack/common` are **lazily resolved**. They point to live Discord modules and are populated when Discord's webpack instance has finished loading — after the `WebpackReady` event fires. Do not access them during module-level initialization or at the top level of a `start()` that runs before `StartAt.WebpackReady`.

<Note>
Most plugins start at `StartAt.WebpackReady` (the default), so by the time `start()` runs, all common exports are available. If your plugin sets `startAt: StartAt.Init` or `startAt: StartAt.DOMContentLoaded`, treat these exports as potentially `undefined`.
</Note>

## React

The `React` object and its most-used hooks are re-exported directly. These are the exact same values Discord itself uses, so JSX from your plugin and JSX from Discord share the same React instance.

```typescript
import {
    React,
    ReactDOM,
    createRoot,
    useState,
    useEffect,
    useLayoutEffect,
    useMemo,
    useRef,
    useReducer,
    useCallback
} from "@webpack/common";
```

| Export | Type | Description |
|--------|------|-------------|
| `React` | `typeof import("react")` | The full React namespace. |
| `ReactDOM` | `typeof import("react-dom")` | React DOM including `createPortal`. |
| `createRoot` | `typeof import("react-dom/client").createRoot` | React 18 `createRoot`. |
| `useState` | hook | Shorthand for `React.useState`. |
| `useEffect` | hook | Shorthand for `React.useEffect`. |
| `useLayoutEffect` | hook | Shorthand for `React.useLayoutEffect`. |
| `useMemo` | hook | Shorthand for `React.useMemo`. |
| `useRef` | hook | Shorthand for `React.useRef`. |
| `useReducer` | hook | Shorthand for `React.useReducer`. |
| `useCallback` | hook | Shorthand for `React.useCallback`. |

## Flux stores

Flux stores are Discord's single-source-of-truth for all client state. Every exported store is populated via `waitForStore` internally, so they are ready as soon as `WebpackReady` fires.

```typescript
import { UserStore, GuildStore, ChannelStore, SelectedChannelStore } from "@webpack/common";

// Get the currently logged-in user
const me = UserStore.getCurrentUser();

// Get a guild by id
const guild = GuildStore.getGuild(guildId);

// Get the channel the user currently has open
const channelId = SelectedChannelStore.getChannelId();
```

### Reacting to store changes

Use `useStateFromStores` to subscribe a React component to one or more Flux stores. The component re-renders whenever the selected value changes.

```typescript
import { useStateFromStores, UserStore, ThemeStore } from "@webpack/common";

function MyComponent() {
    const currentUser = useStateFromStores([UserStore], () => UserStore.getCurrentUser());
    const theme = useStateFromStores([ThemeStore], () => ThemeStore.theme);

    return <div className={`my-component theme-${theme}`}>{currentUser?.username}</div>;
}
```

### `FluxDispatcher`

`FluxDispatcher` lets you dispatch Flux actions and subscribe to them.

```typescript
import { FluxDispatcher } from "@webpack/common";

// Subscribe to an action
FluxDispatcher.subscribe("MESSAGE_CREATE", handler);

// Dispatch an action
FluxDispatcher.dispatch({ type: "SOME_ACTION", payload: data });

// Unsubscribe
FluxDispatcher.unsubscribe("MESSAGE_CREATE", handler);
```

## UI components

Discord's own UI components are re-exported so your plugin renders native-looking UI without shipping its own component library.

```typescript
import {
    Tooltip,
    TextInput,
    Select,
    Slider,
    Checkbox,
    Button,
    Avatar,
    MaskedLink,
    Timestamp,
    ColorPicker
} from "@webpack/common";
```

| Export | Description |
|--------|-------------|
| `Tooltip` | Standard Discord tooltip wrapper. |
| `TextInput` | Single-line text input. |
| `TextArea` | Multi-line text area. |
| `Select` | Dropdown select. |
| `SearchableSelect` | Searchable dropdown. |
| `Slider` | Numeric range slider. |
| `Checkbox` | Toggleable checkbox. |
| `Popout` | Popup positioned relative to an anchor element. |
| `Dialog` | Modal dialog with `role="dialog"`. |
| `TabBar` | Horizontal tab navigation. |
| `Paginator` | Page navigation with prev/next links. |
| `Clickable` | Accessible wrapper for click-handling without a button element. |
| `Avatar` | User avatar image. |
| `ColorPicker` | Color picker with eyedropper support. |
| `MaskedLink` | Hyperlink with Discord's link masking behavior. |
| `Timestamp` | Formatted message timestamp. |
| `FocusLock` | Traps keyboard focus within a container. |
| `UserSummaryItem` | Horizontal list of user avatars. |
| `OAuth2AuthorizeModal` | OAuth2 authorization prompt. |

### Scrollers

Vencord wraps Discord's internal scroller factory to expose ready-to-use components:

```typescript
import { ScrollerThin, ScrollerAuto, ScrollerNone, ListScrollerThin, ListScrollerAuto } from "@webpack/common";

function MyScrollableList({ items }) {
    return (
        <ScrollerThin>
            {items.map(item => <div key={item.id}>{item.name}</div>)}
        </ScrollerThin>
    );
}
```

### Animations

`Animations` exposes Discord's transition primitives built on its animation library:

```typescript
import { Animations } from "@webpack/common";

const { Transition, animated } = Animations;
```

### Legacy `Forms` exports

`Forms.FormTitle` and `Forms.FormText` remain available for backwards compatibility but point to Vencord's `Heading` and `Paragraph` components. Prefer importing those directly.

```typescript
    SnowflakeUtils,  // fromTimestamp, extractTimestamp

    lodash,          // lodash (same instance Discord uses)
```

### Toasts

```typescript
import { showToast, Toasts } from "@webpack/common";

// Quick helper — shows a message toast
showToast("Settings saved", Toasts.Type.SUCCESS);

// Manual form for full control
Toasts.show(Toasts.create("Something went wrong", Toasts.Type.FAILURE, {
    duration: 5000,
    position: Toasts.Position.BOTTOM
}));
```

Toast types: `MESSAGE`, `SUCCESS`, `FAILURE`, `CUSTOM`, `CLIP`, `LINK`, `FORWARD`, `BOOKMARK`, `CLOCK`.

### Alerts

```typescript
import { zustandCreate, zustandPersist } from "@webpack/common";
```
